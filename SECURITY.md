# 安全策略

感谢你帮助提升项目安全性。请在发现安全问题时遵循以下流程。

## 支持范围

- 我们优先处理 `main` 分支与最新发布版本中的安全问题。
- 历史版本可能仅提供有限支持或不再维护。

## 报告方式

- **推荐**：使用 GitHub Security Advisory 私密报告。
- 若无法使用 Security Advisory，请联系仓库维护者进行私下反馈。
- 请避免在公开 Issue 中披露漏洞细节。

## 报告内容

请尽量包含以下信息：

- 复现步骤与影响范围
- 受影响的版本/分支
- 可能的修复建议（如有）

## 响应时间

- 我们会在 7 天内确认收到报告。
- 视复杂度不同，修复时间可能有所变化。

感谢你的负责任披露。
